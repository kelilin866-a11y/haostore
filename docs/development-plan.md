# 分阶段开发计划

## 当前阶段：阶段 5 真实支付接入

本阶段在不破坏第四阶段人工确认付款和发货闭环的前提下，新增真实支付能力。当前实现使用 Stripe Checkout 作为第一版在线支付网关：用户仍然无需注册、无需登录、无需购物车，只在商品详情页选择规格、数量、联系方式和支付方式。

本阶段明确不做：

- 不新增用户注册、登录、会员体系。
- 不新增购物车、批发价、阶梯价、会员价。
- 不替换第四阶段的 `ADMIN_TOKEN` 简易后台保护。
- 不移除人工付款和后台人工确认发货流程。

## 阶段 1：项目初始化 + 前台基础页面

- 初始化 Next.js + TypeScript + App Router。
- 集成 Tailwind CSS 和 shadcn/ui 风格基础组件。
- 预留 Prisma + PostgreSQL 配置。
- 创建首页、产品中心、商品详情、订单查询、订单成功、文章、客服、政策和后台占位页面。
- 使用 mock 数据展示商品、文章和订单。

状态：已完成。

## 阶段 2：数据库模型 + seed 数据

- 设计 AdminUser、Category、Product、ProductVariant、InventoryItem、Order、OrderItem、DeliveryItem、PaymentRecord、ArticleCategory、Article、Setting 模型。
- 添加商品、库存、订单、支付、发货、文章状态枚举。
- 配置 Prisma Client、PostgreSQL datasource、索引和唯一约束。
- 创建基础 seed 数据。

状态：已完成。

## 阶段 3：商品读取 + 下单流程

- 从数据库读取商品和库存状态。
- 商品详情页提供真实规格、数量和联系方式表单。
- 创建待支付订单，但不扣减库存，不创建发货内容。
- 创建 `/order/[orderNo]/pay` 支付说明页。

状态：已完成。

## 阶段 4：后台人工确认付款 + 自动发货

- 实现订单查询闭环。
- 使用 `ADMIN_TOKEN` 保护后台订单页面。
- 管理员人工确认付款后，在事务中扣减库存、创建 DeliveryItem 并完成订单。
- 已发货订单展示发货内容，未发货订单不泄露库存内容。

状态：已完成。

## 阶段 5：真实支付接入

- 新增在线支付方式 `gateway_reserved` 的 Stripe Checkout 支付入口。
- 新增 Checkout Session 创建接口。
- 新增 Stripe webhook 回调接口，用于支付成功后校验签名并更新支付状态。
- 在线支付成功后不自动发货，仍由后台人工确认后再扣减库存并创建 DeliveryItem。
- 保留人工付款、后台人工确认发货和订单查询流程。

状态：当前阶段。

## 阶段 6：SEO 文章系统 + AI 生成草稿

- 实现文章管理、分类、slug、meta、canonical 和 FAQ。
- 接入 AI provider 生成 SEO 文章草稿。
- 管理员审核后发布。

## 阶段 7：支付网关增强

- 补充更多支付服务商。
- 增强支付失败、退款、人工复核和支付审计。
- 根据业务地区评估支付宝、微信、USDT 或其他网关。

## 阶段 8：安全加固和部署

- 增加后台认证、权限、审计日志和基础风控。
- 配置生产环境变量、数据库、部署流水线和监控。
- 完成性能、安全和可用性检查。
