# 虚拟商品自动发货商城需求摘要

## 项目定位

建设一个无登录、无购物车的虚拟商品商城，销售账号类、卡密类和教程类文本商品。用户通过商品详情页直接下单，填写联系方式后生成订单。

## 核心边界

- 无登录：用户不需要注册或登录即可购买和查询订单。
- 无购物车：商品详情页直接下单。
- 不做批发：第一版面向普通零售购买流程。
- 不做评价：不实现商品评价、评分和评论系统。
- 文本库存发货：支付状态确认后，发货仍由后台管理员人工确认，系统再从文本库存中分配发货内容。
- 订单号 + 联系方式查询：用户通过订单号和下单联系方式查询订单。
- Stripe Checkout 在线支付：支付成功后系统通过 Stripe webhook 自动确认付款状态。
- 后台人工确认发货：支付成功前不会展示任何发货内容，管理员确认发货后才展示。
- SEO 文章系统：支持文章列表、文章详情、关键词、meta 信息和相关推荐。
- AI 辅助生成 SEO 文章草稿：后续后台预留 AI 文章生成能力。
- 后台管理商品、库存、订单、文章：后续阶段实现真实后台管理功能。

## 第一阶段范围

第一阶段只完成项目初始化、Tailwind CSS、shadcn/ui 基础组件、Prisma + PostgreSQL 配置预留、前台页面骨架、mock 数据展示、后台静态占位和健康检查 API。

第一阶段历史范围不实现真实支付、不实现用户注册、不实现完整后台业务、不创建真实订单、不分配真实库存。当前项目已接入 Stripe Checkout，发货仍由后台管理员人工确认。
